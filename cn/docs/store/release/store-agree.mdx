请确保开发者后台提交的游戏资料符合《TapTap 游戏审核规范细则》，违规游戏可能被处罚或下架。  

## **一、 基本说明**  
### **1. 法律法规**

1.1 根据国家有关规定，未经国家新闻出版广电总局批准的移动游戏，不得在大陆地区上线运营，即没有版号的游戏不得有任何形式的内购计费内容。

1.2 根据国家有关规定，游戏资质不允许互相套用。如版号出版时间早于软著出版时间，该类游戏将不予以收录。

1.3 游戏不得含有违法内容或其他敏感信息，包括但不限于涉政、涉赌、暴力、血腥、虐待动物或儿童、淫秽、色情、性暗示及广告法中不允许使用的词汇。

1.4 游戏不得带有人身攻击或者侮辱、诽谤他人，侵害他人合法权益（包括但不限于著作权、商标权、肖像权、名誉权等）。

1.5 游戏不得对武器进行过于逼真的表述（如不能涉及武器的制造工艺和参数等），并宣扬违法或滥用武器。

1.6 游戏不得存在反政府、反社会或者不符合主流政治、国家法律禁止的行为。

1.7 游戏不得存在安全隐患、召集、推销、鼓动犯罪、有明显侵犯社会善良风俗行为的、扰乱社会秩序或违规违法嫌疑信息。


### **2. 游戏内付费**

2.1 游戏内所有付费必须明码标价，必须明确告知可享受的服务，如付费后使用的服务与说明不符，游戏可能被下架，严重者会被封禁账号。

2.2 游戏不得存在诱导付费行为，包括但不限于诱导站外直接充值付费、诱导进行线下交易、二维码赞助众筹等。

2.3 游戏不得存在自动扣费等行为。


### **3. 游戏内广告及捆绑下载**

3.1 游戏不得存在空白广告位或者招商广告位，且主要目的不得是展示广告或市场营销。

3.2 游戏广告不得存在模仿系统通知或提示，从而诱导用户点击的行为。

3.3 游戏广告不得在应用已关闭或者退出至后台时依然存在。

3.4 游戏不得存在无法关闭的悬浮窗广告或弹窗广告。

3.5 游戏广告不得存在捆绑下载行为，例如，游戏登录注册页面默认勾选下载其他 APP 或者必须下载其他 APP 才可使用。

3.6 游戏不得存在需强制用户下载其他应用或游戏才可使用。

3.7 广告与其他非广告信息同时出现，可能造成用户误解为内容时，广告应当显著标明「广告」。

3.8 游戏启动页未经用户许可不得默认勾选下载其他 APP ，如存在默认勾选项需明确提示用户。

3.9 游戏未经用户许可不得自动下载其他 APP 。


### **4. 游戏重复**
4.1 请勿上传多个内容相似、相同的游戏，雷同游戏可能会被下架处理。

4.2 游戏的新版本主体功能不得与旧版本的主体功能相差过大，如旧版本为文字游戏，新版本更新为动作游戏。

4.3 游戏不得为简单网站页面打包或套用模板，用户体验质量过低的游戏可能会被下架处理。

4.4 游戏主要功能不得依赖于第三方应用或者需跳转至网页来获取内容及功能。

4.5 游戏内容不得与已经收录的游戏相同，如发现被侵权请进行申诉，点击查看[侵权投诉流程](https://www.taptap.cn/doc/27) 。 


### **5. 欺诈行为**

5.1 游戏存在欺诈、误导用户的行为将被下架，严重者会被封禁账号。

5.2 游戏在审核前后通过服务端控制应用内容，在上架后开启违规服务将被下架，严重者会被封禁账号。


### **6. 暂不收录游戏**

6.1 非官方包，如第三方联运游戏、游戏内含有折扣充值/低价充值等。

6.2 破解、盗版、未获得版权所有者授权或重新打包第三方游戏的行为（包括但不限于图片、音乐和文字素材等）。

6.3 赌博类、博彩类地下彩类（包括但不限于棋牌、捕鱼和娃娃机等）或是带有宣扬赌博色彩的游戏，如使用包含炸金花（扎金花）、梭哈、百家乐、赌场、比大小、赢三张、三张牌、六合彩、轮盘、港式五张、21 点、黑杰克等词汇作为应用的名称；或包含以真实筹码为原型的道具购买内容。

6.4 实物奖励类、现金奖励类或网赚类的游戏。

6.5 虚拟货币类、比特币类或区块链类的游戏。

6.6 色情类、血腥暴力类的游戏。

6.7 内容涉及宗教、民族文化，宣扬邪教和封建迷信，煽动民族仇恨、破坏民族团结的游戏。

6.8 二次创作的同人游戏。

6.9 不具备游戏玩法与游戏属性的应用软件。

6.10 违反国家法律法规的游戏。

## **二、 基础信息**
### **1. 厂商**

1.1 厂商名称仅限使用汉字、数字、字母，及其组合，不得使用特殊符号。  

1.2 厂商名称不得使用占位文本、空格、乱码等无关字符。  

1.3 厂商名称不得使用游戏名命名。  

1.4 厂商名称不得使用个人姓名命名。  

1.5 厂商名称不得使用多个合并或并列的厂商名称。  


### **2. 游戏类型、兼容性及付费**

2.1 游戏类型需根据游戏实际情况填写，不得选择与产品实际功能不符的分类。

2.2 游戏兼容性需根据实际情况选择，不得随意填写，避免误导用户。

---

## **三、 物料审核**
### **1. 基本规范**

1.1 文案、图片及视频中不得含有违反平台收录标准的内容，包括但不限于提现、实质性奖励信息内容及相关素材。

1.2 文案、图片及视频中不得含有与游戏内容无关的素材，包括但不限于其他游戏或应用广告、厂商联系方式。

1.3 文案、图片及视频中不得含有侵权内容，已获授权素材或购买素材需上传相应证明材料或授权书。

1.4 文案、图片及视频中不得含有游戏商店评分等数据信息。

1.5 文案、图片及视频中不得含有与该游戏无关信息。

1.6 文案、图片及视频中不得添加强制引导信息。

1.7 文案、图片及视频中不得涉及游戏评价内容，包括邀请评价、引导好评等。

1.8 文案、图片及视频中不得添加引导至其他网站的信息。

1.9 图片及视频素材必须保证清晰，不得出现明显模糊、拉伸、压缩、黑边、白边等情况，不得使用纯色、渐变等过于简单图案。

1.10 图片及视频如需遮挡裸露角色的敏感部位，对人物进行遮挡请添加衣物，请勿使用“圣光”、“暗幕”、“云团”、黑色遮挡、表情包及马赛克。

1.11 游戏若需求上架中国大陆地区，请提供简体中文版文案与物料。


### **2. ICON 图标**

2.1 ICON 图标不得添加存在误导用户或违反相关规定的素材或角标，不得添加与游戏内容无关的热门搜索词或信息，如极速版、提现等。

2.2 ICON 图标必须符合游戏内容设定，不得添加与游戏内容无关的热门搜索词或信息。


### **3. 详情页截图**

3.1 详情页截图不得含有手机 UI ，或不属于提供安装版本的 UI 。

3.2 详情页截图需保持尺寸基本一致，并且确保图片方向一致，如：如果游戏选择上传横图，请保证所有的图片均为横图。

3.3 详情页截图不得使用重复单一截图。

3.4 详情页截图不得添加存在误导用户或违反相关规定的信息或素材。

3.5 详情页截图需和游戏实际内容一致，不得使用非该游戏内容的素材进行宣传推广。

3.6 详情页截图不得使用翻转、颠倒朝向的图片素材，请使用正向图片。

3.7 详情页截图不得添加商业营销内容。 


### **4. 宣传图​**

4.1 宣传图需含有游戏名，且请勿出现游戏名以外的文字。

4.2 宣传图不得直接使用以未经排版的游戏截图。

4.3 宣传图不得使用多图拼接、平铺的素材。

4.4 宣传图不得出现游戏 ICON 图标素材。

4.5 宣传图不得添加误导信息，如虚假下载按钮。

4.6 宣传图除了以真人为卖点的游戏外，不得出现真人素材。

4.7 宣传图不得出现实物手机。


### **5. 简介及开发者的话**

5.1 简介用以介绍该游戏玩法、特色，请勿添加无关广告信息。

5.2 简介与开发者的话不得违反基本法律法规，需符合基本要求。

5.3 简介与开发者的话不得含有违反平台收录标准的内容。

5.4 简介与开发者的话不得涉及游戏评价内容，包括邀请评价、引导好评等。

5.5 简介与开发者的话不得含有引战、蹭热度等信息。

5.6 简介与开发者的话不得含有引流信息。

5.7 简介与开发者的话不得包含兑换码（包括通码）玩家交流群、除 Steam 平台和 TapTap 详情页链接之外的外部链接。

5.8 简介内容不得可与开发者的话内容高度相似。 


### **6. 游戏标题**
6.1 游戏标题用以填写游戏名，需与游戏资质对应，不得添加热门搜索词类副名称。

6.2 游戏标题中不得添加违反平台规定的信息。

6.3 游戏标题中不得添加误导用户、与游戏内容无关的信息。例如：极速版、重制版、送百抽。

6.4 游戏安装到手机上显示的 App 名称需与游戏详情页展示的游戏标题一致。

6.5 游戏标题中不得含有除 ：，？！~ - · 外的特殊符号。


### **7. 宣传视频、游戏实机录屏**

7.1 视频内不得包含其他平台、剪辑软件的品牌水印。

7.2 视频内不得包含手机 UI，包括模拟手机画中画，及手机状态栏、消息提醒、弹窗等手机 UI。

7.3 视频内不得涉及表情包元素。

7.4 游戏实机录频规范可见[游戏物料要求](https://developer.taptap.cn/docs/store/release/store-material/)。


### **8. 玩家交流群**

8.1 玩家交流群名称需与游戏或官方匹配。

8.2 玩家交流群名称不得带有引流信息。

8.3 玩家交流群名称不得带有其他渠道或平台标识。

8.4 玩家交流群号码不得填写多个号码。

8.5 玩家交流群链接格式为纯链接，请勿添加其他文字或符号。

## **四、 商店配置**
### **1. 游戏状态**

1.1 游戏状态请根据实际情况填写。

1.2 测试服/先行服不支持选择游戏状态，前台默认为「关注」。

1.3 未配置状态地区，游戏状态与「默认」保持一致。


### **2. 预约里程碑**

2.1 预约里程碑内容仅针对游戏正式公测开服。

2.2 预约里程碑支持最大 3 档预约。

2.3 预约里程碑奖励素材尺寸为 200x200px 透明背景，且大小不超过 100 KB 的 PNG 素材。

2.4 预约里程碑奖励素材不得使用黑白图案。

2.5 预约里程碑里程数仅为 TapTap 平台预约数值。

2.6 预约里程碑里程数需使用阿拉伯数字，不得出现任何符号。

2.7 预约里程碑奖励内容需是游戏内具体奖励，奖励内容不得模糊。

2.8 已添加的预约里程碑内容不得自行修改。


### **3. 官网链接**
3.1 需填写游戏官方网站链接，不可填写第三方网站链接。

## **五、 安装包文件**
### **1. 包名**

1.1 包名不得含有任何渠道相关标识。

1.2 包名须与官方包名保持一致。

1.3 请勿使用打包工具默认包名。

1.4 包名作为游戏唯一标识，请确保唯一性。

1.5 包名将用于接入 TapTap 开放平台的游戏服务，提示用户更新等，请确保 APK 包名资料与后台上传 APK 的实际包名一致。 


### **2. 版本号（Version Code）**

2.1 游戏版本号不得为 0。   

2.2 新提交安装包版本号不得低于当前版本号。 


### **3. 版本名（Version Name）**

3.1 版本名需与官网保持一致。  


### **4. 更新日志**

4.1 更新日志内容需与游戏版本变动有关，不得出现宣传、广告或其他无关内容。

4.2 更新日志不得出现联系方式，例如微信、QQ、群聊等。


### **5. 签名**

5.1 APK 不得使用公用证书签名。

## **六、 资质文件及授权**
### **1. 游戏资质**

1.1 上传游戏的公司如与软著或版号的公司主体为子母公司、分公司，请提交真实有效的证明。

1.2 上传游戏的公司如与软著或版号的公司存在授权关系，请提交完整、真实、可追溯的授权链证明。

1.3 上传游戏的公司如有软著或版号中的著作人、运营单位发生变更，请提交真实有效的证明。

1.4 如果存在授权关系，必须提供独家授权书文件。


### **2. 运营和 IP 授权**

2.1 上传游戏的公司如与软著或版号的公司主体为子母公司、分公司，请提交真实有效的证明。

2.2 上传游戏的公司如与软著或版号的公司存在授权关系，请提交完整、真实、可追溯的授权链证明。

2.3 上传游戏的公司如有软著或版号中的著作人、运营单位发生变更，请提交真实有效的证明。

2.4 如果存在授权关系，必须提供独家授权书文件。


### **3. 素材授权**
3.1 游戏素材不得使用版权敏感内容，包括但不限于网络素材、同人素材、其他游戏内素材。

3.2 游戏素材若有真人形象，需提交肖像使用授权书。

3.3 游戏素材若为原创素材，需提交真实有效的证明。

3.4 游戏使用素材若为购买的商业素材，需提交素材购买页面和订单证明。 


### **4. 版号**
4.1 开发者需提供网络游戏出版物号（ISBN）以及核发单扫描件（必须为 JPG 或 PNG 格式），如果提交游戏的厂商非该版号运营单位，需提供相应版号授权，请务必保证授权链独家且完整。

4.2 如版号有更名、转移主体等历史，请务必提交完整的扫描件材料（拼接为同一张，必须为 JPG 或 PNG 格式）。

**特别注意**
- ISBN 号格式必须为 999-9-9999-9999-1
- 如版号有更名、转移主体等历史，请务必提交完整的扫描件材料（拼接为同一张，必须为 JPG 或 PNG 格式）
- 根据国家有关规定，未经国家新闻出版广电总局批准的移动游戏，不得上线运营，即没有版号的游戏不得有任何形式的内购计费内容
- 如果游戏在大陆地区开放预约及信息展示，则暂不需要版号信息
- 如果游戏需要在大陆地区开放下载，请确保资质齐全，如果涉及授权，请提供独代授权


### **5. 软著**

5.1 开发者需提供权利获取方式为「原始取得」的软件著作权登记号以及登记证书扫描件（必须为 JPG 或 PNG 格式），如果提交游戏的厂商非该软著著作权人或权利获取方式为「受让取得」的，需提供相应软著授权，请务必保证授权链独家且完整。

5.2 如软著有更名、转移主体等历史，请务必提交完整的扫描件材料（拼接为同一张，必须为 JPG 或 PNG 格式）。


### **6. APP 的 ICP 备案**

6.1 开发者需要提供 APP 的 ICP备案信息，包括备案号和备案主办单位等信息，以确保应用的合法性和合规性，符合国家互联网信息管理的要求。


### **7. 实名认证与防沉迷**

7.1 实名认证和防沉迷所提交的信息必须真实有效。  

7.2 实名认证和防沉迷的视频需符合[TapTap 实名认证和防沉迷说明](/store/release/store-policy#32-防沉迷政策填写要求/)中的填写要求。
