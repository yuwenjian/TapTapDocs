## 一、手机游戏入库

1. 进入[开发者中心](https://developer.taptap.cn/) >> <Blue>创建游戏</Blue> >> 按照要求填写基础信息 >> 游戏平台选择<Blue>手机</Blue> >> 保存并提交。
2. 创建游戏后，游戏的基础信息已经保存。请在<Blue>商店</Blue> >> <Blue>游戏资料</Blue> >> <Blue>商店资料</Blue>处调整基础信息，如果需要删除或重新创建游戏，请在「全部游戏」面板点击对应游戏右下角的“┆”按钮进行操作。
3. 游戏发布上架请点击游戏图标或游戏标题进入游戏管理面板，选择<Blue>商店</Blue> >> <Blue>游戏资料</Blue> >> <Blue>商店资料</Blue> >> 完善详情资料后提交审核。当发布设置为立即上线时，将在游戏资料审核通过后立即对玩家展示游戏详情页。当发布设置为定时上线时，将在游戏资料审核通过且定时到了之后对玩家展示游戏详情页。

**特别注意**

- 创建游戏后，游戏当前状态为「未发布」状态，此时游戏还未入库，并且游戏详情页不对玩家可见
- 创建游戏后，可在「全部游戏」面板查看创建过的游戏，点击游戏图标或游戏标题即可进入对应游戏的管理面板。厂商也可以点击页面顶部的「厂商名称」或「游戏名称 >>「全部游戏」查看全部已创建的游戏
- 游戏在「未发布」状态下可以使用<Blue>商店</Blue>、<Blue>游戏服务</Blue>和<Blue>数据分析</Blue>的部分功能，<Blue>游戏运营</Blue>的部分功能无法使用
- 游戏预约期间不强制要求上传 APK，但<Blue>游戏服务</Blue>的部分功能需要上传 APK 并通过审核后才可以使用
- 「未发布」状态游戏的上限为 20 个，超过将无法创建新游戏
- APK 包名具有唯一性，是作为游戏唯一标识。当游戏内接入 TapSDK 服务时必填包名，校验移动应用程序备案时也会同时校验 APK 包名

## 二、游戏预约

1. 厂商可根据自己的宣发节点选择游戏状态是「预约」或「敬请期待」，游戏状态在<Blue>商店</Blue> >> <Blue>游戏资料</Blue> >> <Blue>商店资料</Blue>底部导航栏处选择。
2. TapTap 官方建议游戏入库的同时开启预约，玩家会主动选择预约感兴趣的游戏，浏览过详情页的玩家可点击“预约”按钮转化为预约量。当游戏开放测试或正式上线时，系统将给预约玩家推送站内信通知游戏事件。


**特别注意**

- <Blue>创建游戏</Blue>时只能选择「预约」或「敬请期待」，如果游戏需要开启「测试」或「正式上线」，请在<Blue>游戏资料</Blue>处上传 APK ，同时请在<Blue>商店 >> 游戏资料 >>游戏资质</Blue>处完善相关资质并通过审核。

## 三、游戏资质

1. 上传游戏资质请点击「编辑」按钮进行填写并保存。
2. 已保存的资质不会自动提交审核，上传并保存资质后，请点击页面右上角的「提交审核」按钮。如果已提交的信息需要再次修改，可点击「撤销审核」撤回审核。

**特别注意**

- 测试服/先行服仅支持上传官方游戏出版物号（ISBN）、未成年人防沉迷、ICP 备案，如果需要上传其他资质，请前往游戏正式服上传

## 四、游戏更新

1. 在「全部游戏」面板点击进入对应的游戏图标或游戏标题即可进入游戏管理面板，选择<Blue>商店</Blue> >> <Blue>游戏资料</Blue> >> <Blue>商店资料</Blue> >> 完善或更新详情资料后提交审核。
2. 首次创建的游戏在「未发布」状态下且未提交审核时，可以直接完善或修改资料。如果更新的版本已经通过审核，则需要点击<Blue>「+新版本」</Blue>更新资料。

**特别注意**

- 点击<Blue>「+新版本」</Blue>时默认复制上一个版本提交的所有信息，包括 APK 和图片素材
- 在无需更新 APK 的情况下，可以直接修改文字或图片素材并提交审核。如果需要更新 APK，点击<Blue>重新上传</Blue>按钮，然后上传新的 APK，最后提交审核
- 游戏不支持直接变更包名，变更包名会导致已经安装游戏的玩家需要重新安装，可能出现存档丢失的情况。如果需要变更包名，请于[开发者中心](https://developer.taptap.cn/) >> <Blue>工单</Blue>对应分类下提交申请

## 五、游戏测试或上线

1. 游戏如需开启测试，请在<Blue>商店</Blue> >> <Blue>版本发布</Blue> >> <Blue>测试</Blue> >> <Blue>封闭式测试/开放式测试</Blue>进行限量测试或不限量测试。
2. 游戏如需结束测试，请在<Blue>商店</Blue> >> <Blue>版本发布</Blue> >> <Blue>测试</Blue> >> <Blue>封闭式测试/开放式测试</Blue>自行操作关闭测试计划。
3. 游戏如需正式上线，请在<Blue>商店</Blue> >> <Blue>游戏资料</Blue> >> <Blue>商店资料</Blue>中修改游戏状态并提交审核。
4. 游戏如需关闭下载，请在<Blue>商店</Blue> >> <Blue>游戏资料</Blue> >> <Blue>商店资料</Blue>中修改状态为「预约」或「敬请期待」并提交审核。

**特别注意**

- 游戏状态为「未发布」状态时，不支持使用测试功能
- 游戏开启测试或正式上线时，请在变更状态前 24 小时内同步提交 APK 并确保相关游戏资质已经通过审核
- 测试服/先行服不支持选择游戏状态，详情页默认发布状态为「敬请期待」，玩家只可以关注页面
- 游戏下架默认为关闭下载，如果需要下架游戏详情页，请于[开发者中心](https://developer.taptap.cn/) >> <Blue>工单</Blue>对应分类下提交申请

## 六、定时功能

通过<Blue>创建游戏</Blue>和点击<Blue>「+新版本」</Blue>时填写的信息，都可在<Blue>商店 >> 游戏资料>> 商店资料>>发布设置</Blue>中选择「定时上线」或「立即上线」。定时功能可满足以下几种场景：首曝/首测/首发/版更、发布状态变更、新游戏资料（APK 和素材）。

**特别注意**

- 通过<Blue>创建游戏</Blue>新建的游戏版本在首次设置定时上线功能时，需要将时间设置在当前时间的 6 小时后。在审核通过后，可点击<Blue>「修改定时」</Blue>调整定时到当前时间之后的任意时间
- <Blue>游戏资质</Blue>不支持定时功能，为确保游戏能够正常在设置定时上线的时间上线，请务必预留时间提交资质审核，并确保资质审核通过
- 通过<Blue>创建游戏</Blue>新建的游戏版本在首次提交审核后，TapTap 官方预计在 1-3 日操作审核，还请留意定时功能设置的时效范围
- 通过点击<Blue>「+新版本」</Blue>更新的游戏资料在提交审核后，TapTap 官方预计在 1 日内操作审核，还请留意定时功能设置的时效范围

## 七、游戏审核状态及对应的操作

1. 游戏审核状态可在「全部游戏」面板中游戏图标右侧和对应游戏的<Blue>商店资料</Blue>查看，不同审核状态对应的操作有所不同。

**不同游戏资料状态下对应的操作**

| 游戏资料状态 | 操作                                                                                                                                                                               |
| ------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 未发布       | <ul><li>完善资料，提交审核</li><li>删除项目</li></ul>                                                                                                                              |
| 审核中       | <ul><li>查看提审的资料，无法编辑</li><li>撤销审核，撤销后，变更为「未发布」状态</li></ul>                                                                                          |
| 审核失败     | <ul><li>查看审核失败的原因</li><li>编辑资料，再次提交审核</li></ul>                                                                                                                |
| 定时上线     | <ul><li>仅支持查看定时上线的资料，无法编辑</li><li>修改定时上线的时间，仅支持修改为当前时间之后的时间</li><li>撤销定时上线，撤销后，当前资料版本变更为「审核失败」状态</li></ul> |
| 已上线       | <ul><li>查看已上线版本的资料，无法编辑</li><li>构建新版本</li></ul>                                                                                                                |
| 已下架       | <ul><li>查看已下架版本的资料，无法编辑</li><li>构建新版本</li></ul>                                                                                                                |

## 八、预约里程碑

1. 预约里程碑用来帮助游戏吸引玩家进行预约。可在<Blue>游戏运营</Blue> >> <Blue>内容管理</Blue> >> <Blue>预约里程碑</Blue>中设置预约里程碑并提交审核，审核通过后将在游戏详情页展示。
2. 预约里程碑最多三档，需填写内容及要求：
- 达成里程预约数
- 奖励内容：限 25 字符
- 奖励图片：透明背景 200 x 200px，不超过 100KB，格式必须为 PNG
3. 预约里程碑发布后不可自行修改，请确认后再提交审核。如因配置错误或不可抗力需修改预约里程碑，请先在论坛发布预约里程碑修改公告，公告内容需要包括：新达成奖励数、新奖励内容、预约里程碑需修改原因等。公告发布后，请于[开发者中心](https://developer.taptap.cn/) >> <Blue>工单</Blue>中对应的分类下提交修改申请。

## 九、常见问题 Q&A

### 1. TapTap 是所有的游戏都收录吗？

详情可参考[TapTap 游戏审核规范细则](/store/release/store-agree#暂不收录游戏/)。

### 2. 创建游戏审核通过后，可暂不发布游戏吗？

可以，请在<Blue>商店 >> 游戏资料>> 商店资料>>发布设置</Blue>中选择「定时上线」然后提交审核。

### 3. 我的游戏已经被 TapTap 收录，可以进行游戏认领吗？

请于[开发者中心](https://developer.taptap.cn/) >> <Blue>工单</Blue>中对应的分类下提交申请，或邮件发送至运营部邮箱 operation@taptap.com ，并提供游戏资质材料/企业营业执照扫描件/其他相关证明材料。

**【申请格式 】**

> 申请标题：TapTap 游戏认领申请 - XXX 游戏名称
> 
> 需认领游戏名称及页面链接：
> 
> 申请认领的厂商名称及厂商页链接：
> 
> 游戏资质材料（附件上传）：
> 
> 营业执照扫描件或相关证明材料（附件上传）：
> 
> 申请人联系方式：

TapTap 官方人员确认所提交资料合规后，预计在 1-3 个工作日处理。

### 4. 游戏主体可以进行转移吗？

可以。  
请于[开发者中心](https://developer.taptap.cn/) >> <Blue>工单</Blue>中对应的分类下提交申请，并提供游戏原所有方厂商和游戏主体新所有方厂商的授权证明材料。

**【 申请格式 】**

> 申请标题：TapTap 游戏主体转移申请 - XXX 游戏名称
> 
> 需进行主体转移的游戏链接：（如有测试服页面必填）
> 
> 游戏主体原所有方厂商名称或厂商页链接：
> 
> 游戏主体新所有方厂商名称或厂商页链接：
> 
> 转让授权书/资质证明（附件上传）：
> 
> 申请人联系方式：

请确保授权链独家且完整，TapTap 官方人员确认所提交资料合规后，预计在 1-5 个工作日内处理。

### 5. 如何下架游戏详情页？

请于[开发者中心](https://developer.taptap.cn/) >> <Blue>工单</Blue>中对应的分类下提交申请，并提供企业营业执照扫描件或其他能证明厂商官方人员的材料。

**【 申请格式 】**

> 申请标题：TapTap 游戏下架申请 - XXX 游戏名称
> 
> 需下架页面的游戏链接：
> 
> 下架原因：
> 
> 期望下架时间：
> 
> 营业执照扫描件或资质证明材料（附件上传）：

**特别注意**

下架游戏详情页前，请确保游戏状态为「预约」或「敬请期待」，然后说明下架原因。TapTap 官方人员确认信息后，预计在 1-2 个工作日内处理。

### 6. 新提交的 APK 可以回退版本吗？

可以。
请于[开发者中心](https://developer.taptap.cn/) >> <Blue>工单</Blue>中对应的分类下提交申请并声明需要回退的版本号。

### 7.游戏标题可以修改吗？

未获取版号/软著/ICP 备案等资质的游戏请在<Blue>商店</Blue> >> <Blue>商店资料</Blue>中点击<Blue>「+新版本」</Blue>修改标题然后提交审核。

有已上传版号/软著/ICP 备案等资质的游戏无法直接修改游戏标题，请于[开发者中心](https://developer.taptap.cn/) >> <Blue>工单</Blue>中对应的分类下提交申请，然后在<Blue>商店</Blue> >> <Blue>游戏资质</Blue>中将更名后的资质一并提交审核，同时在<Blue>商店</Blue> >> <Blue>商店资料</Blue>中点击<Blue>「+新版本」</Blue>修改标题然后提交审核。

**特别注意**

修改游戏标题时，请统一修改游戏图标、宣传图、游戏截图等资料，游戏标题必须与相关资质全部保持一致。我们建议游戏修改标题后在「开发者的话」或论坛中通知玩家修改游戏标题的原因，以免给玩家带来不良游戏体验，影响游戏评分。


### 8. 预约里程碑内容有限制吗？

预约里程碑需对奖励内容进行清晰说明，不可用例如 “ 预约大礼包 ” 等字样表示。预约里程碑不得涉及任何实物或金钱性质内容（包括且不限于现金、礼品）。

### 9. 如何向玩家发放预约里程碑奖励？

预约里程碑奖励一般在游戏首发后向预约玩家发放，建议在游戏内以站内信/邮件的形式直接向全服玩家发放。

如需通过 TapTap 礼包码的形式发放，请于<Blue>游戏运营</Blue>  >> <Blue>礼包码</Blue> >> <Blue>新增游戏码</Blue>中上传礼包码然后提交审核，建议合并预约里程碑奖励。如需只把礼包码发放给预约玩家，请[开发者中心](https://developer.taptap.cn/) >> <Blue>工单</Blue>中对应的分类下提交申请。

### 10. APK 包名支持修改吗？ 

变更包名会导致已经安装游戏的玩家需要重新安装，可能出现存档丢失的情况。同时，包名将用于接入 TapTap 开放平台的游戏服务，提示用户更新等，如果游戏接入了 TapSDK，请在商店资料位置填写正确的 APK 包名，并确保 APK 包名资料与后台上传 APK 的实际包名一致。

如果需要变更包名，请于[开发者中心](https://developer.taptap.cn/) >> <Blue>工单</Blue>对应分类下提交申请。
