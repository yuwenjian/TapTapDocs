## 功能介绍

**捆绑包** 是一种用于游戏中付费内容管理的工具。为了满足开发者多样化的售卖形式需求，平台上线了捆绑包功能，支持游戏本体（安卓端和 PC 端）、游戏本体与 DLC、DLC 与 DLC、双人捆绑/多人捆绑的销售形式。

:::caution 注意
目前暂不支持同一厂商的不同款付费游戏/DLC 商品进行捆绑销售。
:::

## 如何创建捆绑包

### 配置流程

#### 创建路径

进入 TapTap 开发者中心，选择目标游戏应用，依次进入商店 >> 买断制游戏 >> 捆绑包页面。

![](https://img.tapimg.com/market/images/0558c6f3ff7f214abb9c2a32d9773ff1.png)

#### 前提条件

创建捆绑包前，须确保至少拥有一个 **已通过审核** 的商品（包括付费下载游戏或 DLC ）。商品通过审核后，可选择保持“未上架”状态，仍可用于组建捆绑包。

#### 提交审核

完成捆绑包内容配置与信息填写后，需提交平台审核。审核通过后方可生效。

#### 展示说明

- 捆绑包仅支持在游戏 TapTap 详情页相关区域展示与销售。
- 不支持通过游戏内 SDK 或接口发起购买流程。

### 捆绑包信息填写建议

:::caution 注意
其中 <sup><Red>\* 必填</Red></sup> 标注为必须填写的内容

捆绑包正式上架之前，所有内容均可修改

捆绑包上架之后，类型、内容、上架时间均不支持修改，其余内容可修改
:::

1. ​**捆绑包名称**<sup><Red>\* 必填</Red></sup>

建议按照以下格式配置：游戏名 + 空格 + 游戏本体/完整版/(DLC 名称)DLC +（平台版本）x 数量

举例：

鬼谷八荒 完整版 （Android 版本）x1

火山的女儿：再度重逢（TapTap PC 版）x4

2. ​**捆绑包类型**<sup><Red>\* 必填</Red></sup>

目前只支持选择【必须一并购买】，常用于：游戏本体和 DLC 捆绑成豪华版进行销售。

3. ​**捆绑包内容**<sup><Red>\* 必填</Red></sup>

选择已有的过审的游戏本体以及 DLC 商品进行配置。至少添加 2 个商品，否则无法保存。

![](https://img.tapimg.com/market/images/2641d6fb030c4b6f21813224cb5dbe9c.png)

4. ​**捆绑包封面图** <sup><Red>\* 必填</Red></sup>

格式要求： JPG / PNG 格式，大小在 2MB 以内

尺寸：1920\*1080 px

5. ​**捆绑包内容**​ <sup><Red>\* 必填</Red></sup>

建议介绍捆绑包内具体商品内容，以及支持的版本，会在游戏详情页面展示给玩家。

6. ​**售卖设置**​ <sup><Red>\* 必填</Red></sup>

- 折扣配置
  - 折扣范围可设置为 0 至 100 之间的任意整数值。设置为 0 表示按商品原价组合销售，无折扣。设置为 100 表示捆绑包免费提供。
  - 定价说明：系统将根据捆绑包内各商品的独立售价与所设折扣率，自动计算捆绑包最终售价。例如：若捆绑包内包含两个定价均为 3 元的 DLC 商品，设置折扣为 50%，则捆绑包最终售价为 3 元。
- 上架时间管理
  - 捆绑包默认与所包含商品的最早可售时间保持同步。
  - 若设置的捆绑包开售时间早于其中任一商品的上架时间，则该捆绑包将自动转为预购模式。在商品正式上架前，用户可提前购买捆绑包，待所有包含商品均上架后系统将自动完成发放。
