## 功能概述
云存档支持玩家将游戏存档保存到至 TapTap 云端，使玩家可从任意设备读取存档继续游戏。避免因存档丢失导致玩家流失。

## 服务优势
- 免费
    - 云存档服务对开发者和玩家均免费，仅需按照要求合理控制存档大小和数量。
- 实用
    - 解决单机游戏痛点，极大降低玩家存档焦虑。

## 准备工作
- 接入 TapTap 登录
    - 云存档依赖于 TapTap 账号，因此需玩家先完成 TapTap 登录。

## 使用流程
接入使用排行榜能力整体分3步：
1. 接入 SDK
2. 测试功能
3. 对外发布

### 1.接入 SDK
参照[开发指南]接入 SDK ，调用所需方法即可实现功能。
您应注意云存档的相关限制，否则可能导致云存档失败：
- 单个存档最大10MB
- 单个玩家账号在单个游戏的云存档，最多100个存档文件、100MB总大小
- 两次上传云存档间最小间隔1分钟
因此请通过 SDK 提供的「删除云存档」等方法，合理管理玩家云存档数量及大小。同时注意仅在关键节点进行云存档同步，避免超频调用导致上传失败。

### 2.测试功能
接入完成后，使用自身 TapTap 账号测试即可。

### 3.对外发布
对外发布前，请确认云存档字段已带上必要的信息。添加合适的拓展信息，有助于后续做云存档管理。可参考常见问题中的应用场景。

## 常见问题
- 游戏中何时上传云存档合适？
    - 不同游戏适合的时机不同，可结合游戏实际考虑：
        - 仅自动存档的游戏：产生关键进度存档时上传云存档。
        - 手动存档、读档的游戏：玩家手动存档时上传；或设置「上传云存档」「下载云存档」按钮供玩家按需操作。
    - 无论如何选择上传云存档时机，都请注意：
        - 上传云存档间隔控制在1分钟以上。
        - 针对上传失败给出提示或重试机制。玩家可能由于网络问题等，错过某一次上传时机。游戏可结合实际给予提示或重试，如：存档频繁的游戏，由于很快有下一次上传时机，可无需特别处理；手动上传的游戏，上传失败时给出提示，便于玩家手动重试。
- 单机游戏想要做删档测试，如何清除测试时玩家的云存档？
    - 其中一种做法：从第一次测试开始，在云存档字段中附带版本号等信息。后续新版本游戏添加逻辑：不读取并删除带有旧版本号信息的云存档。
    - 也可灵活利用时间、存档名、存档描述等字段用于综合判断。
    - 为避免玩家仍保留旧版本游戏进行游玩，您还可配套使用更新唤起服务，来实现游戏强制更新。详情参考[更新唤起 - 方式1 - 开发者中心配置更新](/sdk/update/features/)
