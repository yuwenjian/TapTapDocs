## SDK 获取

### 第一步：添加 SDK 所需的外部依赖

SDK 内部使用的 JSON 解析库为 `Newtonsoft-json`，如果当前工程已接入该依赖库，则不需额外处理，否则需在 `Packages/manifest.json` 添加如下依赖：

```json
"com.unity.nuget.newtonsoft-json":"3.2.1"
```

### 第二步：添加 SDK 依赖

SDK 支持远程依赖 (Unity Package Manager) 及本地文件导入两种集成方式，开发者根据需求选择其中一种即可，推荐使用远程依赖。

#### 远程依赖

SDK 支持通过 NPMJS 及 GitHub 两种方式，开发者选择其中一种即可。

##### 1. NPMJS 接入


在项目的 `Packages/manifest.json` 文件中添加以下依赖：


```json
"dependencies":{
    "com.taptap.sdk.core":"4.5.6",
    "com.taptap.sdk.login":"4.5.6"
}
```


但需要注意的是，需在 `Packages/manifest.json` 中 `dependencies` 同级下声明 `scopedRegistries`：


```json
"scopedRegistries":[
  {
    "name": "NPMJS",
    "url": "https://registry.npmjs.org/",
    "scopes": ["com.taptap"]
  }
]
```


##### 2. GitHub 接入

在项目的 `Packages/manifest.json` 文件中添加以下依赖：

```json
"dependencies":{
    "com.taptap.sdk.core":"https://github.com/taptap/TapSDKCore-Unity.git#4.5.6",
    "com.taptap.sdk.login":"https://github.com/taptap/TapSDKLogin-Unity.git#4.5.6"
}
```

在 Unity 顶部菜单中选择 **Window > Package Manager** 可查看已经安装在项目中的包。

#### 本地文件导入

1. 在 [下载页](/tap-download) 下载下列模块对应 `unitypackage` 文件，并在 Unity 项目中依次通过 **Assets > Import Packages > Custom Packages** 进行导入，包括：

  <ul>
    <li><code>TapSDK_Core.unitypackage</code> TapTapSDK 核心模块，必选</li>
     <li><code>TapSDK_Login.unitypackage</code> TapTapSDK 登录模块，必选</li>
  </ul>

2. 如果当前项目已集成 `Newtonsoft.Json` 依赖，则忽略该步骤，否则在 `NuGet.org` [ Newtonsoft.Json ](https://www.nuget.org/packages/Newtonsoft.Json) 页面中通过点击右侧 「Download package」 下载库文件，并将下载的文件后缀从`.nupkg` 修改为 `.zip`，同时解压该文件并复制内部的 `Newtonsoft.Json.dll` 文件拷贝到工程 `Assets` 的 `Plugins` 目录下，另外为了避免导出 IL2CPP 平台时删除必要数据，需在 `Assets` 目录下创建 `link.xml` 文件（如果已有该文件，则添加如下内容），其内容如下：

```xml
<linker>
  <assembly fullname="System.Core">
    <type fullname="System.Linq.Expressions.Interpreter.LightLambda" preserve="all" />
  </assembly>
</linker>

```

## 初始化

根据 TapTap 开发者中心的应用配置信息，调用如下接口完成初始化

```cs
using TapSDK.Core;

// 核心配置
TapTapSdkOptions coreOptions = new TapTapSdkOptions
{
   // 客户端 ID，开发者后台获取
   clientId = "游戏 clientId",
   // 客户端令牌，开发者后台获取
   clientToken = "游戏 clientToken",
   // 地区，仅支持 CN 国内
   region = TapTapRegionType.CN,
   // 是否开启日志，Release 版本请设置为 false
   enableLog = true,
   // 客户端公钥，开发者后台获取
   clientPublicKey = "游戏 clientPublicKey"
};
// TapSDK 初始化
TapTapSDK.Init(coreOptions);
```

## 校验游戏启动方式

为了保证游戏能够通过 TapTap 客户端获取必要的应用和授权信息，开发者在调用初始化接口后，必须调用该接口来完成前置启动方式校验，当返回值为 true 时，游戏可进行后续登录等其他业务流程，当返回结果为 false 时，SDK 会通过弹窗的形式引导用户重新通过 TapTap 客户端启动游戏或下载 TapTap 客户端，游戏需停止进行后续业务流程 。

由于校验过程属于异步操作，所以调用该接口前建议游戏先展示 Loading 弹窗， 当 SDK 返回结果后，再关闭 Loading 弹窗。调用示例如下：

```cs
using TapSDK.Core;

bool IsSuccess = await TapTapSDK.IsLaunchedFromTapTapPC();
if (IsSuccess)
{
    UnityEngine.Debug.Log(" TapTap PC 客户端校验通过");
    // TODO: 继续后续登录等其他业务流程
}
else
{
    UnityEngine.Debug.Log(" TapTap PC 客户端校验未通过");
    // 停止执行后续业务流程
}
```



## 登录及获取用户信息

每次发起登录且成功后，SDK 会将登录信息缓存在本地，开发者可先判断本地是否存在有效的登录信息，如果有，直接进入游戏，否则重新发起登录。

通过 TapTap 客户端进行登录授权是无 UI 的异步操作， 建议游戏在发起授权前展示 Loading 弹窗，当收到 SDK 返回结果后，再关闭 Loading 弹窗。调用示例如下：

```cs
using TapSDK.Login;
using System.Threading.Tasks;

// 获取本地用户信息
TapTapAccount taptapAccount = await TapTapLogin.Instance.GetCurrentTapAccount();
// 判断本地用户信息是否有效
if (taptapAccount != null && taptapAccount.accessToken != null){
    // 用户已登录，获取用户信息，进入游戏
    UnityEngine.Debug.Log($"当前已经登录了: {taptapAccount.openId}");
} else {
    // 重新发起授权登录
    try
    {
        // 定义授权范围
        List<string> scopes = new List<string>
        {
            TapTapLogin.TAP_LOGIN_SCOPE_PUBLIC_PROFILE
        };
        // 发起 Tap 登录
        var currentAccount = await TapTapLogin.Instance.LoginWithScopes(scopes.ToArray());
        UnityEngine.Debug.Log($"登录成功，当前用户 ID：{currentAccount.unionId}");
    }
    catch (TaskCanceledException)
    {
        UnityEngine.Debug.Log("用户取消登录");
    }
    catch (Exception exception)
    {
        UnityEngine.Debug.Log($"登录失败，出现异常：{exception}");
    }

}
```

### 不同的授权范围

TapTap 授权登录接口支持选择不同的授权范围且支持任意组合，但必须包含 `public_profile` 和 `basic_info` 中的一个，不然无法获得 openId 和 unionId。TapTap 授权登录接口可以传递 String 类型的数组作为参数，开发者可以根据自己业务需求添加不同的权限为数组的元素。

<Conditional region='cn'>

| 权限 | 说明
|---|---|
| `public_profile` | 获得 openId、unionId、用户昵称、用户头像
| `user_friends` | 获得访问 TapTap 好友相关数据的权限
| `basic_info` | 获得 openId 和 unionId

</Conditional>

### 获取用户信息

```cs
using TapSDK.Login;
using System.Threading.Tasks;

TapTapAccount taptapAccount = await TapTapLogin.Instance.GetCurrentTapAccount();
AccessToken accessToken = taptapAccount.accessToken;
string openId = taptapAccount.openId;
```
其中 `taptapAccount` 可能会包含如下信息：

  | 参数      | 说明                                                                                                                         |
  | --------- | ---------------------------------------------------------------------------------------------------------------------------- |
  | `accessToken` | 玩家在 TapTap 平台的用户 token 信息
  | `name`    | 玩家在 TapTap 平台的昵称                                                                                                     |
  | `avatar`  | 玩家在 TapTap 平台的头像 url                                                                                                 |
  | `openId`  | 通过用户信息和游戏信息生成的**用户唯一标识**，每个玩家在每个游戏中的 `openId` 都是不一样的                                   |
  | `unionId` | 通过用户信息加上厂商信息生成的用户唯一标识，一个玩家在同一个厂商的所有游戏中 `unionId` 都是一样的，不同厂商下 `unionId` 不同 |


`openId` 和 `unionId` 使用[标准的 Base64](https://datatracker.ietf.org/doc/html/rfc4648#section-4)（带 Padding）编码，包含的字符有 `A-Za-z0-9+/=`。`openId` 和 `unionId` 长度最大值为 50 个字符。

:::info
由于 `unionId` 与游戏所属厂商有强关联性，因此 `unionId` 适用于如下场景：厂商使用测试服进行付费删档等测试，正式服需要对于之前参与测试的老玩家进行返利等操作。因为同一个玩家在同一个厂商下的所有游戏中的  `unionId` 不变。
一个游戏在厂商转移后同一个用户的 `unionId` 会发生改变，如果游戏使用了 `unionId`，TDS 技术支持会在转移前通过工单和游戏开发者确认相关数据的处理方案，保证迁移前后用户数据不错乱。
:::



## 登出

登出将清除本地登录信息及缓存

```cs
using TapSDK.Login;

TapTapLogin.Instance.Logout();
```

## 开发调试设置

当游戏尚未发布时，开发者可以在游戏可执行文件所在目录下创建一个名为 `taptap_client_id.txt` 的文本文件。该文件内容仅需填写当前应用的 `clientId`，即可在调试阶段**无需通过 TapTap 客户端启动游戏**，直接运行可执行文件进行调试。

:::tip
注意：当发布游戏时，请务必删除游戏内 `taptap_client_id.txt` 文本文件
:::
