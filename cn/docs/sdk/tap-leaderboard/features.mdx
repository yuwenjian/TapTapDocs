## 功能概述
TapTap 排行榜功能支持开发者自定义维度排行榜。游戏上报成绩，TapTap 完成成绩计算和排序后，提供排行榜界面或数据供游戏展示、分享。
同时在 TapTap 内也将提供游戏排行榜查看、点赞、放榜通知等配套服务，帮助游戏拉新促活。

![](https://img.tapimg.com/market/images/8bec8b54536ee967452b6f3c91a4b460.png)

## 服务优势
- 建立比拼机制，提高玩家活跃
    - 积分、战力值、通关时长…均可作为排行维度，促进玩家积极竞争。
- 社交互动能力，强化荣誉感与社交传播
    - 排行榜UI自带点赞、分享炫耀功能，为每一个高分喝彩。所有排行自带好友榜，与 TapTap 好友一争高下。
- 单机游戏无需额外备案，低成本拥有排行榜系统
    - 单机游戏使用排行榜服务，也无需申请联网游戏备案，无需承担服务器成本。
- 接入简单，功能全面
    - 配备现成版排行榜界面，游戏只需调用一个方法，即可展示功能丰富的排行榜，无需自行绘制界面。同时也提供排行数据方法，供自行绘制界面灵活使用。

## 准备工作
- 接入 TapTap 登录
    - 排行榜依赖于 TapTap 账号，因此需玩家先完成 TapTap 登录。
- 阅读服务引导，开始使用排行榜服务
![](https://img.tapimg.com/market/images/64a0dfb22972a1bc7e007bbe9bcf6fa2.png)

## 使用流程
接入使用排行榜能力整体分4步：
1. 配置排行榜
2. 接入 SDK
3. 测试功能
4. 对外发布

### 1.配置排行榜
![](https://img.tapimg.com/market/images/78ffb17e7e0894961f1939a41798bdb0.png)
创建排行榜时需注意的配置项：
- 排行榜所属分组：分组用于聚合同类排行榜，建议排行榜较多时配置。需先在「创建排行榜」按钮旁的「分组配置」里创建分组。
- 界面设置：所配图片将作为排行榜底图，同时其他UI元素也将从图片中自动取色。未配置时会取游戏资料中配置的「游戏宣传图」作为底图。
- 部分配置项创建后不可修改，请谨慎配置，包括：重置周期、重置时间、数据类型、数据排序方式、数据更新策略、枚举值（仅限枚举数据类型）。
- 其他注意项请参考配置界面中，各项后的小问号提示。
![](https://img.tapimg.com/market/images/6e708d6d963a9bb3848af236b17906b9.png)

创建的排行榜默认开启「仅白名单可见」，便于测试。请前往「测试白名单」添加测试用户。
![](https://img.tapimg.com/market/images/21f731d4e81c07d197d96942c6cb2e43.png)

您可将游戏中最重要的排行榜设置为「默认排行榜」，便于玩家在多处界面里，优先展示此排行榜数据。

### 2.接入 SDK
创建排行榜后会获得对应「排行榜 ID」，结合开发指南接入 SDK ，调用所需方法即可实现功能。

### 3.测试功能
- 若游戏已上线，请务必开启「仅白名单可见」进行测试，避免玩家看见。即使开启「仅白名单可见」，非白名单用户也可上报成绩。
- 通过排行榜的数据管理，可对排行榜成绩数据进行查询、删除、屏蔽。
![](https://img.tapimg.com/market/images/2a4f5071f40b5290c778edfa2e7510fc.png)

### 4.对外发布
正式发布前请检查：
- 排行榜的文案、界面图等是否已是正式配置
- 客户端调用的排行榜ID是否正确
- 排行榜测试数据是否已清除（从数据管理中可清空数据，请注意如果是重置排行榜，请确认所有周期的数据都已清除）
- 排行榜是否关闭「仅白名单可见」

发布上线后，针对成绩异常的用户，您可以：
- 对于数值型、时间型排行榜，可配置「允许最高数值」「允许最低数值」，超出允许范围的成绩将在当期排行榜中被屏蔽。但请注意，该配置适合存在理论极值的情况，若允许范围太小，将导致正常获得的优秀成绩被错误屏蔽，这种情况易发生在累加计算的排行榜，其成绩随着游戏时间可能变得很大。
- 在数据管理中，手动屏蔽或删除异常的成绩。一般建议屏蔽，被屏蔽的用户，当期排行榜无论如何更新成绩，都不会在排行榜中显示，并且可在「本期已屏蔽」中查看记录；而删除成绩后，该用户后续更新成绩依旧会出现在排行榜。
- 将用户加入黑名单，其在当前及之后周期的所有排行榜中，成绩均会被屏蔽。过去已结束的周期，成绩不受影响。如有需要，可手动屏蔽成绩。
- 解除黑名单后，若想重新展示当前或过去周期的成绩，可手动解除屏蔽。
