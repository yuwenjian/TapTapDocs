## 功能概述
云存档支持玩家将游戏存档保存到至 TapTap 云端，使玩家可从任意设备读取存档继续游戏。避免因存档丢失导致玩家流失。

## 服务优势
- 免费
    - 云存档服务对开发者和玩家均免费，仅需按照要求合理控制存档大小和数量。
- 实用
    - 解决单机游戏痛点，极大降低玩家存档焦虑。

## 接入流程
- 小游戏接入
    - 参照[小游戏云存档开发指南](https://developer.taptap.cn/minigameapidoc/dev/api/open-api/cloudsave/tap.getCloudSaveManager/)接入即可。
- Android、iOS、PC 端 SDK 正在白名单测试中，即将推出

您应注意云存档的相关限制，否则可能导致云存档失败：
- 单个存档最大 10MB
- 单个玩家账号在单个游戏的云存档，最多 100 个存档文件、100MB 总大小
- 两次上传云存档间最小间隔 1 分钟

因此请通过「删除云存档」等方法，合理管理玩家云存档数量及大小。同时注意仅在关键节点进行云存档同步，避免超频调用导致上传失败。

向云存档元数据的存档名称、存档描述、额外信息中添加合适信息，有助于云存档管理。可参考下文中具体应用场景。

## 常见问题
- 游戏中何时上传、下载云存档合适？
    - 不同游戏适合的时机不同，可结合游戏实际考虑：
        - 仅自动存档的游戏：产生关键进度存档时上传云存档。
        - 手动存档、读档的游戏：玩家手动存档时上传；或设置「上传云存档」「下载云存档」按钮供玩家按需操作。
    - 无论如何选择上传云存档时机，都请注意：
        - 上传云存档间隔控制在1分钟以上。
        - 针对上传失败给出提示或重试机制。玩家可能由于网络问题等，错过某一次上传时机。游戏可结合实际给予提示或重试，如：存档频繁的游戏，由于很快有下一次上传时机，可无需特别处理；手动上传的游戏，上传失败时给出提示，便于玩家手动重试。
        - 必要时先比对同一个存档在本地与云端的游戏进度、保存时间等数据，再决定是否覆盖本地存档、云存档，或提示玩家自行选择覆盖方式。
- 单机游戏想要做删档测试，如何清除测试时玩家的云存档？
    - 其中一种做法：从第一次测试开始，在云存档元数据的额外信息字段中附带版本号。后续新版本游戏添加逻辑：不读取并删除带有旧版本号信息的云存档。
    - 还可灵活利用存档时间、存档名称、存档描述、额外信息等字段用于综合判断。
