## 功能说明

直达礼包为用户提供方便快捷的礼包领取体验，降低厂商礼包被刷风险。

## 直达礼包

### 场景介绍

用户在 TapTap 客户端（展示位置包括游戏详情页/签到页面/礼包专区等）完成领取动作后，游戏厂商在游戏内直接为该用户帐号的指定区服/角色发放礼包内容，不需要用户先领取礼包串码再复制使用。直达礼包应用场景如下图所示。

### 限制说明

- 目前仅支持接入开启了 TapTap 开发者应用配置的联网游戏。[开发者中心配置](/sdk/access/get-ready/)
- 目前仅支持接入「[TapTap 登录](/sdk/taptap-login/features/)」或「[游戏角色查询](/sdk/game-record/features/)」的游戏应用。
- 在 TapTap 开发者中心配置的该功能服务地址需为备案过的域名且支持 HTTPS，当开发者的服务域名变更时，请务必提前自行变更。

:::info 注意

- 如果您的游戏正在进行站内活动，您将无法开启/关闭直达礼包的功能。站内活动包括论坛抽奖/签到/详情页礼包/H5 抽奖等其他站内 H5 活动；
- 直达礼包功能开启后，您可以随时在开发者服务后台修改直达礼包配置。通过后台的功能自测后，您更新的配置将发布上线。
  :::

## 业务步骤

### 准备工作

1.在接收请求前，需要先 [开启应用配置](/sdk/access/get-ready/) ，以获取到用于请求签算的 `Client ID` 和 `Server Secret`

2.选择接入方式（以下两种方式**二选一**）
- **[TapTap 登录](/sdk/taptap-login/features/)** （由 TapTap 调用提供的角色获取接口，获取玩家的角色信息）
- **[游戏角色查询](/sdk/game-record/features/)** （基于 TapTap「游戏角色查询」能力，获取用户在该服务中已绑定的角色信息）

3.需要在该业务配置后台中增加调用地址配置，包含

- 礼包发送接口地址 **(必填)**
- 区服列表获取接口 **(当接入方式为「TapTap 登录」时选填，多区服游戏根据需要配置)**
- 角色获取接口 **(当接入方式为「TapTap 登录」时必填)**

参考 [直发礼包接口文档](/sdk/instant-gift/guide/) 和 [接口公共说明](/sdk/common/api)

### 测试联调

1.开发者提供服务 URL 地址
     - 当接入方式为 **「TapTap 登录」** 时：提供用于获取玩家区服、角色信息和发放游戏内礼包的接口地址
     - 当接入方式为 **「游戏角色查询」** 时：仅需提供用于发放游戏内礼包的接口地址

2.开发者提供测试礼包码

3.开发者可以在 TapTap 开发者中心自助验证

4.调试接口服务是否正常
