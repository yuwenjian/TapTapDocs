## 产品介绍

游戏角色查询为开发者提供标准化的游戏数据展示解决方案。通过配置游戏素材和提供数据接口，即可在 TapTap 平台为玩家打造专属角色页，展示角色信息、实时角色数据等，让玩家在 TapTap 社区记录和分享游戏成果。

## 接入流程

### 阶段一：账号绑定接入

**参与人员**：运营/研发

**主要任务**：

1. 前往【开发者中心 - 游戏服务 - 应用配置】开启游戏角色查询功能权限
2. 选择账号绑定方式（支持 Weblink、Deeplink 等多种方案）
3. 完成账号绑定接口开发与测试

**开发指引**：详见 [游戏角色账号绑定](/sdk/game-record/server-guide/role-bind/)

### 阶段二：搭建角色查询详情页、提供素材

**参与人员**：运营/设计/研发/项管

**主要任务**：

1. 完成数据推送内部合规流程（游戏侧需要将已绑定游戏角色的数据推送至TapTap侧）
2. 评估游戏角色查询功能上线时间（建议随首发、大版更）
3. 基于角色信息模块、组件搭建角色查询详情页样式（参考[页面搭建](/sdk/game-record/guide-page-editor)）
4. 确认需要展示的数据类型及字段
5. 提供配套素材（角色查询宣发图、统计页素材、角色查询绑定协议）

### 阶段三：接口开发与自测

**参与人员**：研发

**主要任务**：

1. 阅读开发指引，下载推送接口示例（需角色查询详情页搭建完成且素材过审）
2. 根据接口示例规范，实现游戏服务端数据接口
3. 完成数据推送自测（建议使用测试账号验证）

:::tip
新增/调整模块、组件会影响接口示例，若只调整素材则不影响
:::

**开发指引**：详见 [游戏角色数据推送](/sdk/game-record/server-guide/role-data-upload/)

### 阶段四：上线提审

**参与人员**：运营

**主要任务**：

1. 完成角色查询落地页配置并提审
2. 完成角色查询详情页上线时间配置并提审

### 阶段五：后续维护

**参与人员**：运营/研发/设计

**主要任务**：

1. 游戏有**玩法、角色、场景、物品等新增内容**时需要更新游戏角色查询详情页并提审（建议至少版更前1天完成）
2. 重新配置**新角色查询落地页、设定上线时间**并提审

:::tip
新增/调整角色信息模块、组件会影响接口，需要重新下载示例并开发；如仅更新素材则不影响接口
:::
